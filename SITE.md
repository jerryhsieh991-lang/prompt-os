# prompt-os website (`build_site.py` → `site/`)

A static, dependency-free educational website generated **from the loop library itself**
(`loops/*.md`). It presents the 112 loop prompts as a browsable, searchable, teachable
site — the vertical slice **Home → Library → Prompt detail → Anatomy**.

## Build & run

```bash
python3 build_site.py            # parses loops/*.md → writes site/ (161 pages, stdlib only)
cd site && python3 -m http.server 8199    # serve (needed: Library fetches data/prompts.json)
# open http://127.0.0.1:8199/
```

`site/` is generated output — re-run `build_site.py` after editing any `loops/*.md`; the site
stays in sync with the source of truth. Nothing is hand-written per prompt.

## What's on it

| Route | What |
|-------|------|
| `index.html` | Home — hero, a live color-coded anatomy of one real prompt, starter set, family grid, the "one idea". |
| `library.html` | All 112 prompts. Client-side search + filters (family, verifier type, model tier, starter). No server. |
| `prompt/<id>.html` | 112 detail pages. Tabs: **Prompt** (copy-able, loop steps, 4-arm stop, model routing) · **Anatomy** (color-coded segments) · **Why it works** (mechanism, tied to principles) · **Source** (provenance/licensing). |
| `anatomy.html` | The universal loop anatomy: 5 components, the 11 principles, 15 antipatterns — all parsed from `loops/00-loop-engineering-principles.md`. |
| `family/<key>.html` | 14 family pages + the curation near-duplicate notes. |

## How it works (architecture)

- **Parser** reads each `loops/<family>.md` into structured records (`When/Loop/Stop/Model` +
  the fenced prompt body). Stop conditions are anchor-sliced into the 4 arms
  (`SUCCESS/BUDGET/NO-PROGRESS/BLOCKED`) — robust to `:`/`—` separators and the Chinese
  `self-critique` family.
- **Anatomy engine** segments each prompt by *inline anchor labels* (`GOAL (frozen):`,
  `VERIFIER:`, `LOOP (each turn):`, `Carry forward`, `Stop on…`), so both blank-line and
  dense single-paragraph prompts decompose correctly. Every prompt's **frozen goal** and
  **multi-armed stop** are detected in-body (112/112) — the library's core thesis, made visible.
  Titles like `中文(English)` surface the English as the display title.
- **Facets** are *honest, derived* signals — verifier type (mechanical vs judge), model-tier
  hint, length, starter membership. Deliberately **no fabricated 0–100 quality score.**
- Output is plain HTML + one CSS file (light/dark, semantic anatomy colors, accessible —
  every color is paired with a text label) + one small JS file (search, tabs, copy). Fully
  self-contained: no tracking, no external requests, deployable to any static host / GitHub Pages.

## Intentionally out of scope (migration path, not built)

This is the scoped first slice, **not** the full "PROMPT ATLAS" spec. Deferred by design:
`/learn` course · `/patterns` · `/loops` encyclopedia · `/automation` library · `/lab`
analyzer · `/compare` · `/playground` builder · `/research` observatory · `/models` ·
model-run evaluation · a database · authentication · automated ingestion. The parsed
records are already JSON, so a later move to Next.js + Postgres (if the corpus outgrows
static generation) is a clean lift — no rework of the data layer.

## Provenance

All content originates from this repository's loop library, generated by a 32-agent
research → adversarial-verify → generate → curate pipeline and human-reviewed. The site
preserves that provenance on every prompt's **Source** tab; it does not claim ownership or
present reconstructions as originals.
