# Translation / Localization

`translation-localization` — 2 loop prompts.

### 1. Document Translation Verified by Blind Back-Translation / Independent Bilingual Alignment

- **When:** You must translate a document and *prove* meaning survived — not trust the translator's own "this is faithful" — so an independent pass (blind back-translation by a different model/reviewer, or a separate bilingual aligner) reconstructs meaning from the target alone and every divergence is fixed one segment at a time.
- **Loop:** assess the segment carrying the worst unresolved meaning-divergence flag -> re-translate (or split) exactly ONE segment -> verify by BLIND back-translation with a different model/pass (target->source, no sight of the original) + mechanical entity/number/date/negation diff, OR an independent bilingual reviewer -> commit the segment if its divergences clear with none re-introduced elsewhere, else revert -> decide
- **Stop:** SUCCESS: every segment passes the independent meaning check with zero unresolved divergences on a full clean pass · BUDGET: `<MAX_TURNS>` turns / `<MAX_SEGMENTS_TOUCHED>` re-translations / `<MAX_HOURS>` elapsed · NO-PROGRESS: one segment's divergence set is unchanged for `<K>` re-translation attempts despite changing strategy (restructure vs. reword vs. de-idiomatize) · BLOCKED: the source itself is ambiguous/self-contradictory, no faithful rendering exists without altering the source, or the independent verifier is unavailable
- **Model:** The forward translator can be a strong translation-capable model, but it must NEVER decide faithfulness — it knows the intended meaning, so it rationalizes its own output as faithful and shares its own blind spots. The verifier must be independent: the back-translator is a *different* model/family (or a separate context) that sees ONLY the target segment — not the source, not the forward translation's notes — so a reconstructed source that diverges is evidence meaning actually drifted, not a shared assumption. Run the entity/number/date/negation diff as deterministic code (extraction + set-compare), reserving model judgment for propositional meaning. Escalate to a human `<BILINGUAL_REVIEWER>` for high-stakes segments (legal/medical/safety) and to break back-translation-vs-translation ambiguity — round-trips can coincidentally mask a real error or introduce their own.

```text
GOAL (frozen — do not redefine mid-loop)
Translate <SOURCE_DOC> from <SOURCE_LANG> into <TARGET_LANG>, producing <TARGET_DOC> over the frozen <SEGMENT_SET> (unit = <SEGMENT_UNIT>, e.g. one sentence or one bounded paragraph), such that EVERY segment preserves source meaning under <INDEPENDENT_CHECK>: either blind back-translation by <BACK_TRANSLATOR> (a different model/family or separate pass that sees ONLY the target segment) whose reconstructed source aligns to the original within <DIVERGENCE_RUBRIC> — no added or dropped propositions, no negation/quantifier/scope flip, and named entities + numbers + dates + units identical — OR sign-off by <BILINGUAL_REVIEWER>, a human fluent in both who is not the translator. Register and formality must match <REGISTER_SPEC> (e.g. tu/vous, keigo level, honorifics). Freeze before turn 1 and never mutate: the source doc (immutable), both languages, the segmentation, the divergence rubric + its pass threshold, the register spec, and the verifier mechanism. The translator never grades its own output; "faithful" means the same thing on turn 1 and turn 20 — do not relax the rubric, drop a hard clause, or lower the register to declare victory.

INDEPENDENT VERIFIER
The verifier reconstructs meaning from the TARGET ALONE and compares it to the frozen source: <BACK_TRANSLATOR> re-translates the target segment back to <SOURCE_LANG> with zero sight of the original source or the forward translation's reasoning, then a fixed diff (deterministic for entities/numbers/dates/units, rubric-scored for propositions/negation/scope) flags divergences; for high-stakes segments <BILINGUAL_REVIEWER> reviews source-vs-target directly. It cannot rubber-stamp itself because (a) it is a different model/human than the translator and shares none of its intended-meaning context — so a divergence reflects real drift, not a shared assumption; and (b) the entity/number/negation diff is arithmetic/set membership over the two texts, not a "reads faithfully" opinion. Attribution guard: a flagged divergence can originate on the back-translation side, so before blaming the translation, re-run the back-translation via a second independent pass/model; if the reconstruction stabilizes and still diverges, the translation is at fault. Round-trip caveat: a back-translation can coincidentally "repair" a genuine error and produce a false PASS — high-stakes segments therefore require the human reviewer, never back-translation alone.

PER-TURN SHAPE
1. ASSESS — from the last verifier run, pick the single segment with the most severe unresolved divergence (meaning flip and dropped/added proposition outrank stylistic nits). Do not chase scattered cosmetic differences the rubric does not flag.
2. ONE ACTION — make exactly one reversible change to <TARGET_DOC>: re-translate that one segment with a different strategy (restructure clause order, de-idiomatize, disambiguate a term against context), or split one over-long segment into verifiable units. Never edit two segments in one turn; never touch <SOURCE_DOC>, the rubric, or the register spec.
3. VERIFY — run <INDEPENDENT_CHECK>: hand only the revised target segment to a back-translator that has not seen the source, run the mechanical entity/number/date/negation diff, rubric-score propositional alignment, and route high-stakes segments to <BILINGUAL_REVIEWER>. Re-check neighboring/shared segments for any newly introduced divergence.
4. DECIDE — commit the segment (version the doc) if its divergences cleared AND no other segment regressed; otherwise revert to the last committed version and mark that strategy tried-and-failed for this segment. End every turn on a committed, self-consistent target document.

CARRY-FORWARD STATE (compact)
Goal + frozen rubric/threshold/register; current target-doc version; per-segment divergence status (clean / open flags + type); divergence-count trend over last 3 turns; strategies already tried per open segment (don't repeat); which segments are high-stakes (need human, not back-translation); back-translator/reviewer availability; budget remaining (turns, re-translations, hours).

ACTION BAN
Never let the forward translator judge its own faithfulness — that is self-grading. Never let the back-translator see the source or the forward translation's notes (that contaminates it into a rubber stamp). Never edit or "clarify" <SOURCE_DOC> to make a segment pass — the source is fixed ground truth. Never silently disambiguate an ambiguous or self-contradictory source; surface it. Never accept a lone back-translation "match" for a high-stakes segment without the second/human verifier. Never re-translate a segment verbatim after a fail (no A->B->A oscillation) — the next attempt must change approach. Never relax <DIVERGENCE_RUBRIC>, drop a hard clause, or downshift <REGISTER_SPEC> to cross the line.

STOP — halt on the FIRST of:
SUCCESS (every segment passes <INDEPENDENT_CHECK> with zero unresolved divergences on a full clean pass, high-stakes segments human-confirmed) | BUDGET (<MAX_TURNS> turns, <MAX_SEGMENTS_TOUCHED> re-translations, or <MAX_HOURS> elapsed) | NO-PROGRESS (one segment's divergence set unchanged for <K> attempts despite switching strategy — escalate that segment to a human translator rather than keep re-wording) | BLOCKED (the source is ambiguous/contradictory so no single faithful rendering exists, a required meaning cannot be preserved without altering the source, or <BACK_TRANSLATOR>/<BILINGUAL_REVIEWER> is unavailable — surface to a human; never substitute the forward translator's self-assessment as a stand-in verifier)
```

### 2. UI-String Localization Gated by a Mechanical Placeholder/Length/Glossary Linter

- **When:** You must localize UI strings where a dropped `%s`, a missing plural category, a 3-column overflow, or a translated brand term is a runtime crash or layout break — so "done" is decided by a deterministic linter (placeholder parity, length budget, frozen glossary/termbase, plural completeness, pseudo-localization), never by a human or model eyeballing the output.
- **Loop:** assess the string key with the most linter violations -> re-translate/fix exactly ONE key -> run the mechanical linter (ICU/printf placeholder parity, length budget, frozen glossary + do-not-translate, CLDR plural-category completeness, pseudo-localization overflow) -> commit the key if its violations clear with none introduced in shared or plural variants, else revert -> decide
- **Stop:** SUCCESS: every string passes all linter gates with zero violations on a full clean run · BUDGET: `<MAX_TURNS>` turns / `<MAX_STRINGS_TOUCHED>` re-translations / `<MAX_HOURS>` elapsed · NO-PROGRESS: one key's violation set is unchanged for `<K>` re-translations despite different phrasings · BLOCKED: a required glossary term is longer than the key's length budget, source placeholders are malformed, the glossary self-contradicts, or the target locale needs a plural category the catalog's message format can't express
- **Model:** Run the translation itself on a mid/standard tier — the linter, not reasoning depth, decides done, and placeholder-preserving translation of short strings rarely needs a frontier model. Escalate to a stronger model only for keys that *stall*: natural phrasing inside a tight length budget while keeping every placeholder and the mandated glossary term. The verifier must be CODE, not a model: the failure modes (a missing `%s`, an off-by-one plural category, a 2-column overflow, a brand term quietly translated) are exactly what a self-reviewing model skims past — a model grading its own strings shares the precise blind spot that dropped the placeholder. Mechanical checks are runtime/layout facts, not opinions.

```text
GOAL (frozen — do not redefine mid-loop)
Localize the string catalog <STRING_CATALOG> (keys + source values + per-key metadata: placeholder inventory, <LENGTH_BUDGET> per key/UI context, required plural/select categories) from <SOURCE_LOCALE> into <TARGET_LOCALE>, producing <TARGET_BUNDLE>, such that EVERY key passes <LINTER> with zero violations across all gates: (1) PLACEHOLDER PARITY — the target carries the exact same set and count of placeholders as the source: ICU ({name}, {count, plural, ...}, {sel, select, ...}), printf (%s, %d, %1$d), %@, {{var}}, and markup/HTML tags — none invented, none dropped, positional indices correct if word order is reordered; (2) LENGTH — target length <= <LENGTH_BUDGET> measured in <LENGTH_METRIC> (grapheme clusters / display columns, not code units, accounting for CJK width); (3) GLOSSARY — every term in <GLOSSARY> rendered as its mandated <TARGET_LOCALE> term, every <DNT_LIST> (do-not-translate/brand) term left verbatim, no forbidden term present, matched on token boundaries not substrings; (4) PLURAL/SELECT COMPLETENESS — all CLDR categories <TARGET_LOCALE> requires are present (e.g. Polish one/few/many/other, Arabic zero/one/two/few/many/other, Japanese other) — not merely the source's categories; (5) PSEUDO-LOCALIZATION — the target and a pseudo-expanded source render in the UI slot with no truncation/overflow/clipping, and escapes (\n, \t) and leading/trailing whitespace are preserved. Freeze before turn 1 and never mutate: the catalog + source values, both locales, the length budgets + metric, the glossary + DNT list, the linter ruleset, and the required plural categories. "Done" is the linter's zero-violation verdict, identical on turn 1 and turn 20 — never a human "looks fine."

INDEPENDENT VERIFIER
The verifier is <LINTER>, deterministic code that reads ONLY the artifacts and returns pass/violations per key: a set comparison of source-vs-target placeholders (with index/plural-branch awareness), a measurement of rendered length against <LENGTH_BUDGET>, a membership/token lookup against <GLOSSARY> and <DNT_LIST>, a CLDR category set-check, and a render/overflow pass over the UI slot including pseudo-localized expansion. It cannot rubber-stamp itself because it is not the translator and makes no "reads naturally" judgment — every gate is a mechanical fact (a placeholder is present or absent, a string fits or overflows, a term is verbatim or not, a category exists or is missing). A model reviewing its own strings shares the exact blind spot that dropped the %s or the `few` branch; only the linter reliably catches the runtime-crash / layout-break class. The linter runs after every single-key change, over that key AND every string that shares its placeholders or plural message, so a fix can't silently break a sibling variant.

PER-TURN SHAPE
1. ASSESS — from the last full linter run, pick the single key with the most severe violation set (missing placeholder and missing plural category outrank a length overflow, which outranks a stylistic nit). Note its gate failures.
2. ONE ACTION — make exactly one reversible change: re-translate or repair that one key's value (or fill in a missing plural/select branch for that key). Keep placeholders intact and, when reordering is needed, switch to indexed placeholders (%1$s) rather than dropping or duplicating. Never edit two keys in one turn; never touch the catalog, budgets, glossary, DNT list, or linter ruleset.
3. VERIFY — run <LINTER> on the changed key and every string sharing its placeholders/plural message: placeholder parity, length under <LENGTH_METRIC>, glossary + DNT, CLDR plural completeness, and pseudo-localization overflow. A zero-violation result on that key with no new violation elsewhere is the only pass.
4. DECIDE — commit the key (version the bundle) if its violations cleared AND no sibling regressed; otherwise revert to the last committed value and mark that phrasing tried-and-failed for this key. End every turn on a committed bundle that is no worse than the previous.

CARRY-FORWARD STATE (compact)
Goal + frozen budgets/metric/glossary/required-categories; current bundle version; per-key gate status (clean / which gates failing); total open-violation count trend over last 3 turns; phrasings already tried per stalled key (don't repeat); keys that share placeholders or a plural message (fix together / re-check together); budget remaining (turns, re-translations, hours).

ACTION BAN
Never edit <STRING_CATALOG>, <LENGTH_BUDGET>, <GLOSSARY>, <DNT_LIST>, or the linter ruleset to make a string pass — that is moving the goalposts. Never drop, merge, or invent a placeholder because the target sentence "doesn't need it" — parity is exact. Never leave source-locale text untranslated to fit the budget, and never fix an overflow by silently truncating. Never omit a plural/select category <TARGET_LOCALE> requires, and never translate a <DNT_LIST>/brand term. Never accept "it renders fine to me" over the linter — the linter is the verifier. Never re-run the identical translation after a fail (no A->B->A oscillation) — the next attempt must change phrasing or structure. Never smuggle two keys' edits into one turn (you lose attribution). Never lower a budget, weaken a glossary rule, or disable a linter gate to declare done.

STOP — halt on the FIRST of:
SUCCESS (every key passes all five linter gates with zero violations on a full clean run of <LINTER> over <TARGET_BUNDLE>) | BUDGET (<MAX_TURNS> turns, <MAX_STRINGS_TOUCHED> re-translations, or <MAX_HOURS> elapsed) | NO-PROGRESS (one key's violation set unchanged for <K> re-translations despite different phrasings/structures — escalate that key to a stronger model or a human localizer rather than keep re-wording) | BLOCKED (a mandated <GLOSSARY> term is physically longer than the key's <LENGTH_BUDGET>, the source placeholder inventory is malformed, <GLOSSARY> contradicts itself or the DNT list, or <TARGET_LOCALE> requires a plural category the catalog's message format cannot express — surface to the UI/glossary owner for a budget change, catalog fix, or redesign; never silently drop a placeholder, overflow the budget, or omit a category to force a pass)
```
